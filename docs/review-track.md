# Review Track

The register is useful only if its entries are checked. `Review wanted` means an
entry is relevant enough to track, but one or more checks are still open.

## Review Types

- Source review: confirm the project, dataset, paper, or disclosure URL.
- Licence review: check code licence, data licence, reuse rights, and
  attribution requirements.
- Fit review: check whether the item belongs under the stated problem.
- Evidence review: check openness level and evidence strength.
- Safety review: check market-integrity, sensitive-data, and operational-risk
  caveats.

## Small Reviews Are Welcome

A review can be narrow. For example:

- "The repository URL is correct, but the data licence is unclear."
- "This is open code, but not a reproducible benchmark."
- "This belongs under EP-901, but only as modelling infrastructure."
- "The public disclosure is relevant, but the underlying data cannot be reused."

## How To Submit

Use the review request issue template or open a pull request. Include:

- card ID;
- review type;
- source checked;
- conclusion;
- caveats.

Maintainers decide whether a review changes `review_status`, openness level,
evidence strength, or problem status.

Open energy register

Open Energy Problems

A Germany-first public register of energy-market and grid problems, tracking how far each has an open benchmark, schema, dataset, simulator, replay package, source ledger, validated method, or public disclosure.

GitHub repository Problem cards Tracked references Scope review

Why It Exists

The German grid is becoming a stress test for software-defined energy: storage economics, congestion, balancing markets, demand response, asset data, and flexibility claims all need better public evidence.

This register starts with those problems and asks what open references already exist, what they prove, and what is still missing.

How To Read It

Green means a useful reference exists for a stated scope. Yellow means partial progress or public disclosure. Red means no accepted open reference is listed yet.

The table is deliberately conservative: it tracks inspectable evidence, not hype, vendor claims, or live trading logic.

Status

  • Open: problem stated, no accepted open reference.
  • Partial: meaningful progress, but important gaps remain.
  • Reference available: useful public reference exists.
  • Solved for scope: open reference exists for a narrow stated scope.

Open-Solution Level

  • 0: no accepted open reference.
  • 3: public methodology.
  • 4: open code.
  • 5: reproducible benchmark.
  • 6: open reference infrastructure.

Evidence

  • A: open code, data, reproducible result, independent review.
  • B: open code, documented method, sample data.
  • C: public method or figures, limited reproducibility.
  • D: public claim or unclear detail.
